Previous releases

AVR Assembler 2.0 Released with AVR Studio 4.11 Jan-Mar 2005

  • AVRASM2 is now the default AVR Studio assembler. A lot of bugs are fixed, and some new features added, based on feedback from users testing the beta releases.

New include files and XML Converter.

  • A complete new set of include files are now available for AVRASM2. These should resolve most of the issues experienced during the AVRASM2 beta period.

  • The XML converter program used to machine-generate the new include files is included in this release. In addition to generating assembler include files, this program can be used to produce header files for AVR-GCC, and header and linker files for the IAR C compiler.

AVR Assembler v1.77.3 date Mar-2005

  • The old assembler and include files are still included in this AVR Studio release. It may be selected in Studio for use with existing projects, but is now unsupported and support for new devices will not be added. This will be the last AVRASM1 version.

AVR Assembler v1.77.1 date Oct-2004

AVR Assembler 2 Beta date Jun-2004

  • A new AVR assembler version with many new features: AVR Assembler 2

 

AVR Assembler v1.76 date May-2004

  • Included device directive for  ATmega165.

  • Detect and report incorrect use of conditionals on forward referenced labels, causing incorrect offset calculations for relative branches.

  • Instruction set issues for some parts fixed.

  • Assembler recognizes '#' as comments (same as ';'), to cause it ignore AVRASM2 preprocessor directives.

AVR Assembler v1.75 date Mar-2004

  • Included device directives for ATmega88, ATmega168, ATmega329 and AT90CAN128.

  • DSEG directive is now correct for devices with extended IO

  • Wrap relative jumps.  For devices with more than 4k words of program memory, using this option may cause unpredictable results and it should be turned off.  If it is left on, the assembler will now produce a warning.

  • Assembler now warns about operand combinations that cause undefined behaviour, like 'ld r30,z+'

  • Assembler now supports extended ASCII characters in character constants and strings.

  • Undetected syntax errors in .db directive fixed
